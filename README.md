# project-3
This is for SI568 project 3.
Our team is building an intentionally unethical data science plan called Vulnerability Fusion Engine. The idea is to combine three kinds of public data signals, car insurance risk, health insurance cost burden, and loneliness or social disconnection, to show how an unethical company could build a broader profile of who is most vulnerable to unfair pricing, worse coverage, and manipulative treatment. The project is not about directly linking real people across all three datasets. Instead, we use separate public datasets as stand-ins for different forms of vulnerability and show how cross-domain profiling could be used in harmful ways. This fits the project goal because it follows the stages of a data science project while clearly violating ethical and best-practice standards through privacy invasion, bias amplification, unfair targeting, and misleading storytelling.
